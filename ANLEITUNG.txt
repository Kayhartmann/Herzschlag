HERZSCHLAG – PROJEKT (PWA + Capacitor-Android-App)
====================================================

Was hier drin ist:
- apk-aktualisieren.bat -> Doppelklick erledigt Sync + Build automatisch
- www/index.html      -> aktueller Spielstand (Firebase, Bestenliste,
                          Profil, Codes, News-Banner, responsive Overlays)
- www/manifest.json    -> PWA-Manifest (App-Name, Icons, Farben)
- www/sw.js            -> Service Worker fürs Offline-Caching
- www/icon-192.png,
  www/icon-512.png     -> einfache Platzhalter-Icons
- www/three.min.js + weitere .js -> Three.js r128 + Bloom-Postprocessing
- package.json, capacitor.config.json -> für den Android-App-Build


ERSTMALIGE EINRICHTUNG (nur beim allerersten Mal nötig)
-----------------------------------------------------------
Vorher installieren, falls noch nicht vorhanden:
- Node.js:        https://nodejs.org/
- Java JDK 17:     https://adoptium.net/
- Android Studio:  https://developer.android.com/studio

Im Terminal, im entpackten Projektordner:

   npm install
   npm install @capacitor/core @capacitor/cli @capacitor/android
   npx cap add android


★★★ WICHTIGSTER SCHRITT BEI JEDER ÄNDERUNG DER index.html ★★★
--------------------------------------------------------------------
Reines Überschreiben von www/index.html reicht NICHT aus – Capacitor
kopiert den Inhalt von www/ separat in den android/-Ordner rein. Ohne
den Sync-Befehl baut Gradle weiterhin die ALTE, bereits kopierte
Version.

EINFACHSTER WEG: Doppelklick auf "apk-aktualisieren.bat" im
Projekt-Hauptordner. Die Datei macht automatisch:
  1) npx cap sync android
  2) gradlew.bat assembleDebug
  3) zeigt dir am Ende, wo die fertige APK liegt

Manuell geht es genauso, in dieser Reihenfolge:

1) Neue index.html nach www/index.html kopieren (überschreiben)

2) Im Terminal, im PROJEKT-HAUPTORDNER (nicht im android-Unterordner!):
   npx cap sync android

   -> Das kopiert www/ frisch nach android/app/src/main/assets/public/
      Ohne diesen Schritt ändert sich an der App NICHTS, egal wie oft
      du index.html überschreibst.

3) Erst DANACH in den android-Ordner wechseln und bauen:
   cd android
   gradlew.bat assembleDebug

4) Fertige APK liegt unter:
   android\app\build\outputs\apk\debug\app-debug.apk

5) Vor dem Installieren auf dem Handy: alte Version der App erst
   DEINSTALLIEREN, dann die neue APK installieren. Einfaches
   "Drüberinstallieren" hält manchmal alte zwischengespeicherte
   Web-Inhalte zurück.


WEG A: Als PWA (installierbare Web-App, kein Play Store nötig)
-----------------------------------------------------------------
1) Den kompletten www/-Ordner irgendwo mit HTTPS hosten, z.B. kostenlos
   über GitHub Pages (Repository erstellen, Ordnerinhalt hochladen,
   unter Settings -> Pages aktivieren)
2) Die URL im Handy-Browser öffnen
3) Chrome/Android zeigt automatisch "Zum Startbildschirm hinzufügen" an
4) Fertig – läuft dann wie eine eigene App, mit Icon, ohne Adressleiste


WICHTIG: Firebase (Bestenliste, Codes, News-Banner) braucht immer eine
echte Internetverbindung, egal ob als PWA oder als App. Grafik, Sound,
Spielstand-Speicherung funktionieren weiterhin komplett offline.


GEPLANT, ABER NOCH NICHT UMGESETZT: NATIVES GOOGLE-LOGIN
-----------------------------------------------------------------
Aktuell gibt es einen Wiederherstellungscode (Einstellungen -> Cloud-
Backup) als einfache Lösung, um den Spielstand bei Geräteverlust
zurückzuholen. Echtes Google-Login kommt später als eigener Umbau -
braucht ein natives Plugin (z.B. @capacitor-firebase/authentication)
statt der normalen Firebase-Web-Methode, weil Google seit 2023 aktiv
OAuth-Logins innerhalb von eingebetteten WebViews blockiert
("disallowed_useragent"). Zusätzlich nötig: SHA-1-Zertifikat-Eintrag
in der Firebase-Konsole für die Android-App.


VIBRATION REPARIERT (falls du von einer älteren Version kommst)
-----------------------------------------------------------------
Vibration lief bisher nicht, weil das noetige Capacitor-Haptics-Plugin
nie tatsaechlich installiert war (in package.json gefehlt). Das ist
jetzt behoben und wird beim naechsten Ausfuehren von
"apk-aktualisieren.bat" automatisch mitinstalliert (Schritt 1 fuehrt
jetzt "npm install" aus).

Zur Erinnerung: Android fragt fuer Vibration KEINE Erlaubnis per
Dialog ab - das ist normal und richtig so. VIBRATE zaehlt zu den
"normalen" (ungefaehrlichen) Berechtigungen und wird automatisch beim
Installieren der App gewaehrt, ganz ohne Nutzer-Dialog. Wenn Vibration
nach diesem Update immer noch nicht geht, deinstalliere die alte
App-Version komplett vom Handy, bevor du die neue APK installierst.

Eigene Icons: icon-192.png und icon-512.png sind nur Platzhalter,
gerne durch eigene quadratische PNGs (192x192 / 512x512) ersetzen,
Dateinamen beibehalten.
